    ROS2 対応

    開発中

    ROS2対応は現在開発中です。インターフェースや設定は今後変更される可能性があります。

    概要

    togikaidriveの各センサーモジュールをROS2ノードとして起動し、標準的なROSトピックでデータを配信できます。これにより、SLAM・ナビゲーション・経路計画などROS2エコシステムとの連携が可能になります。

    graph LR
    subgraph togikaidrive
        CAM["camera.py"]
        IMU["imu.py"]
        US["ultrasonic.py"]
        OF["opticalflow.py"]
        LID["lidar.py"]
        RPM["rpm_sensor.py"]
    end
    subgraph "ROS2 Topics"
        T1["&#47;camera&#47;image_raw"]
        T2["&#47;imu&#47;data"]
        T3["&#47;sonner&#47;*&#47;range"]
        T4["&#47;opticalflow&#47;data"]
        T5["&#47;lidar&#47;scan"]
        T6["&#47;rpm&#47;data"]
    end
    CAM -->|"--ros"| T1
    IMU -->|"--ros"| T2
    US  -->|"--ros"| T3
    OF  -->|"--ros"| T4
    LID -->|"--ros"| T5
    RPM -->|"--ros"| T6

    各モジュールは --ros フラグなしでも従来どおりスタンドアロンで動作します。ROS2がインストールされていない環境でも既存機能に影響はありません。

    ROS2 インストール

    Jetson Orin Nano — ROS2 Humble

    Jetson Orin NanoはUbuntu 22.04ベースのため、ROS2 Humble Hawksbill（LTS, 2027年5月EOL）を使用します。

    # ロケール設定
sudo apt update && sudo apt install locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

# ROS2リポジトリ追加
sudo apt install software-properties-common
sudo add-apt-repository universe
sudo apt update && sudo apt install curl -y
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
  -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
  http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | \
  sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

# インストール
sudo apt update
sudo apt install ros-humble-desktop

    環境設定

    # .bashrc に追加
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
source ~/.bashrc

    動作確認

    ターミナルを2つ開いて以下を実行します。

    # ターミナル1: パブリッシャー
ros2 run demo_nodes_cpp talker

    # ターミナル2: サブスクライバー
ros2 run demo_nodes_py listener

    Hello World メッセージが送受信されれば成功です。

    センサーメッセージパッケージ

    togikaidriveのROS2ノードで使用するメッセージ型をインストールします。

    sudo apt install ros-humble-sensor-msgs ros-humble-std-msgs ros-humble-geometry-msgs

    センサーモジュール ROS2 ノード一覧

    すべてのセンサーモジュールは --ros フラグで ROS2 ノードとして起動できます。

    モジュール ノード名 トピック メッセージ型 周期
    camera.py camera_node /camera/image_raw sensor_msgs/Image 30fps
    imu.py imu_node /imu/data sensor_msgs/Imu 10Hz
    ultrasonic.py ultrasonic_node /sonner/<name>/range sensor_msgs/Range 10Hz
    opticalflow.py opticalflow_node /opticalflow/data std_msgs/Float32MultiArray 10Hz
    lidar.py lidar_node /lidar/scan sensor_msgs/LaserScan 10Hz
    rpm_sensor.py rpm_node /rpm/data std_msgs/Float32MultiArray 10Hz

    起動方法

    各モジュールを個別にROS2モードで起動します。

    # LiDAR
python lidar.py --ros

# IMU
python imu.py --ros

# 超音波センサー
python ultrasonic.py --ros

# RPMセンサー（I2Cモード）
python rpm_sensor.py --ros
python rpm_sensor.py --ros --mode gpio   # GPIOモード

# カメラ
python camera.py --ros

# オプティカルフロー
python opticalflow.py --ros

    トピック確認

    # アクティブなトピック一覧
ros2 topic list

# 各トピックのデータ確認
ros2 topic echo /lidar/scan
ros2 topic echo /imu/data
ros2 topic echo /rpm/data

# 配信レートの確認
ros2 topic hz /lidar/scan

    設計方針

    各モジュールは以下の共通パターンで実装されています。

    # ROS2が利用可能な場合のみノードクラスを定義
try:
    import rclpy
    from rclpy.node import Node
    ...
    class SensorNode(Node):
        ...
    def main_ros():
        ...
except ImportError:
    rclpy = None

# --ros フラグで起動モードを切り替え
if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument('--ros', action='store_true')
    args = parser.parse_args()
    if args.ros and rclpy:
        main_ros()
    else:
        main()
    • ROS2未インストール環境では rclpy = None となり、既存動作に影響なし
    • --ros フラグなしでは従来どおりスタンドアロン実行
    • 各ノードは create_timer による一定周期パブリッシュ

    SLAM Toolbox による地図生成

    開発中

    SLAM Toolbox 連携は計画段階です。

    概要

    SLAM Toolbox はROS2標準の2D SLAMパッケージです。LiDARスキャンデータ（/lidar/scan）とオドメトリから2D占有格子地図（Occupancy Grid Map）を生成します。

    graph LR
    LID["lidar.py --ros"] -->|"&#47;lidar&#47;scan"| ST["SLAM Toolbox"]
    IMU["imu.py --ros"] -->|"&#47;imu&#47;data"| ODOM["オドメトリ推定"]
    RPM["rpm_sensor.py --ros"] -->|"&#47;rpm&#47;data"| ODOM
    ODOM -->|"&#47;odom"| ST
    ST -->|"&#47;map"| MAP["2D占有格子地図"]
    ST -->|"&#47;tf"| TF["座標変換"]

    インストール

    sudo apt install ros-humble-slam-toolbox

    sudo apt install ros-jazzy-slam-toolbox

    動作モード

    モード 用途 説明
    Online Async リアルタイム地図生成 走行しながら地図を生成。教育・デモ向け
    Online Sync 高精度地図生成 フレーム落ちなしで処理。低速走行向け
    Offline 記録データから地図生成 rosbagの再生から地図を構築
    Localization 自己位置推定のみ 既存地図上での位置推定

    基本的な起動

    # オンライン非同期SLAM（リアルタイム地図生成）
ros2 launch slam_toolbox online_async_launch.py

# 生成した地図を保存
ros2 run nav2_map_server map_saver_cli -f ~/maps/course_map

    自己位置推定（Localization）

    事前に生成した地図を読み込み、走行中の自己位置をリアルタイムに推定します。

    ros2 launch slam_toolbox localization_launch.py \
  map_file_name:=~/maps/course_map \
  use_sim_time:=false

    Nav2 によるナビゲーション

    開発中

    Nav2 連携は計画段階です。

    概要

    Nav2（Navigation2） はROS2の標準ナビゲーションスタックです。地図上での目標地点への自律移動を実現します。

    graph TB
    subgraph "入力"
        MAP["&#47;map"]
        ODOM["&#47;odom"]
        SCAN["&#47;lidar&#47;scan"]
        GOAL["目標地点"]
    end
    subgraph "Nav2"
        PLAN["Global Planner"]
        CTRL["Local Controller"]
        REC["Recovery"]
        BT["BehaviorTree"]
    end
    subgraph "出力"
        CMD["&#47;cmd_vel"]
    end
    MAP --> PLAN
    GOAL --> PLAN
    PLAN --> BT
    ODOM --> CTRL
    SCAN --> CTRL
    BT --> CTRL
    BT --> REC
    CTRL --> CMD

    インストール

    sudo apt install ros-humble-navigation2 ros-humble-nav2-bringup

    sudo apt install ros-jazzy-navigation2 ros-jazzy-nav2-bringup

    主要コンポーネント

    コンポーネント 役割 説明
    Planner Server グローバル経路計画 地図全体から目標までの経路を計算
    Controller Server ローカル経路追従 障害物を回避しながら経路を追従
    Recovery Server リカバリー動作 スタック時の回転・後退などの回復行動
    BT Navigator 行動管理 BehaviorTreeで各コンポーネントを統合制御
    Costmap 2D コストマップ 静的地図+動的障害物を統合した移動コスト地図

    基本的な起動

    # Nav2起動（事前にSLAMまたは地図読み込みが必要）
ros2 launch nav2_bringup navigation_launch.py \
  map:=~/maps/course_map.yaml \
  use_sim_time:=false

    経路計画アルゴリズム

    開発中

    経路計画の統合は計画段階です。

    グローバル経路計画

    Nav2のPlanner Serverで使用可能な経路計画アルゴリズムです。

    アルゴリズム 特徴 適用場面
    NavFn (A*) 最短経路探索。Nav2デフォルト 一般的な経路計画
    Theta* 任意角度の経路生成。滑らかな経路 広い空間での移動
    Smac 2D 改良型A*。高速計算 リアルタイム性重視
    Smac Hybrid-A* 車両の旋回制約を考慮 ミニカーのような非ホロノミック車両

    ローカル経路追従

    コントローラサーバーで使用可能なアルゴリズムです。

    アルゴリズム 特徴 適用場面
    DWB Dynamic Window Approach。Nav2標準 一般的な障害物回避
    Regulated Pure Pursuit 曲率ベースの追従制御 滑らかな経路追従
    MPPI Model Predictive Path Integral 高度な障害物回避
    Rotation Shim 回転優先のコントローララッパー 大きな方向転換時

    MPC（Model Predictive Control）

    開発中

    MPC制御の統合は計画段階です。

    概要

    MPC（モデル予測制御）は、車両の動力学モデルを用いて将来の状態を予測し、最適な制御入力を計算する手法です。

    graph LR
    STATE["現在の状態"] --> MPC["MPC コントローラ"]
    REF["参照経路"] --> MPC
    MPC --> OPT["最適化問題を解く"]
    OPT --> CMD["制御指令"]
    CMD -->|"最初の1ステップのみ適用"| CAR["車両"]
    CAR -->|"次のタイムステップ"| STATE

    動力学モデル（Kinematic Bicycle Model）

    ミニカーの運動は自転車モデルで近似します。

    状態変数 記号 説明
    位置X \(x\) グローバル座標X
    位置Y \(y\) グローバル座標Y
    ヨー角 \(\psi\) 車両の向き
    速度 \(v\) 前進速度
    制御入力 記号 説明
    ステアリング角 \(\delta\) 前輪操舵角
    加速度 \(a\) 加減速

    評価関数

    MPCは以下のコスト関数を最小化します。

    コスト項 目的
    経路追従誤差 参照経路からの横偏差・ヨー角偏差を最小化
    速度追従誤差 目標速度との差を最小化
    制御入力コスト ステアリング・加速度の大きさを制限
    制御入力変化コスト 急激なステアリング変化を抑制（滑らかな操作）

    Nav2 MPPI Controller

    Nav2にはMPPI（Model Predictive Path Integral）コントローラが含まれており、MPCの一種として利用できます。

    # Nav2パラメータでMPPIを指定
controller_server:
  ros__parameters:
    controller_plugins: ["FollowPath"]
    FollowPath:
      plugin: "nav2_mppi_controller::MPPIController"
      time_steps: 56
      model_dt: 0.05
      batch_size: 2000
      vx_max: 0.5
      vy_max: 0.0
      wz_max: 1.9

    MPCC（Model Predictive Contouring Control）

    開発中

    MPCC制御の統合は計画段階です。

    概要

    MPCC（モデル予測コンタリング制御）は、MPCを拡張した手法です。経路追従だけでなく進行度（経路上の進み具合）も最適化することで、レース走行のようなタイムアタックに適した制御を実現します。

    MPCとMPCCの比較

    項目 MPC MPCC
    目的 経路への正確な追従 最速でのコース走破
    評価関数 横偏差 + 制御コスト 横偏差 + 制御コスト + 進行度
    速度制御 目標速度への追従 コーナーで減速、直線で加速を自動判断
    適用場面 安全な自律走行 レース・タイムアタック
    計算コスト 中 やや高

    MPCCの評価関数

    MPCCではコスト関数にコンタリング誤差とラグ誤差を分離し、進行度パラメータを導入します。

    コスト項 記号 説明
    コンタリング誤差 \(e_c\) 経路に対する横方向の偏差（小さくしたい）
    ラグ誤差 \(e_l\) 経路上の最近接点からの遅れ（小さくしたい = 前に進みたい）
    進行度報酬 \(-v_\theta\) 経路上の進行速度（最大化 = 速く進みたい）
    制御コスト \(u\) ステアリング・加速度の滑らかさ

    F1TENTH チームによるMPCC実装

    MPCCはF1TENTH（1/10スケール自律レーシング）コミュニティで広く採用されています。以下は代表的なオープンソース実装です。

    ETH Zurich — Alexander Liniger（原論文実装）

    MPCCの原点となる実装。1:43スケールRCカーでの自律レースを実現した論文に基づきます。

    項目 内容
    リポジトリ alexliniger/MPCC
    言語 C++ / MATLAB
    ソルバー hpipm（時変QP近似）
    論文 "Optimization-Based Autonomous Racing of 1:43 Scale RC Cars" (2015)
    • 状態にインテグレータ（経路上の進行度 \(\theta\)）を追加
    • ラグ誤差（縦）とコンタリング誤差（横）をコスト関数で分離
    • MATLAB版は障害物回避にも対応

    ForzaETH — ETH Zurich（F1TENTH国際大会優勝チーム）

    F1TENTH国際大会で複数回優勝したチームのフルレーシングスタック。

    項目 内容
    リポジトリ ForzaETH/race_stack
    大学 ETH Zurich, D-ITET Center for Project-Based Learning
    論文 "ForzaETH Race Stack" (Journal of Field Robotics, 2025)
    • 認識・状態推定・計画・制御・対戦相手推定をモジュール化
    • Head-to-Head（1対1対戦）レースに対応
    • 市販ハードウェア（COTS）のみで構成

    Nonlinear MPCC — Niraj Basnet（Jetson対応）

    非線形MPCCをROS + CasADiで実装。NVIDIA Jetson TX2 での動作実績あり。

    項目 内容
    リポジトリ nirajbasnet/Nonlinear_MPCC_for_autonomous_racing
    言語 Python / ROS
    ソルバー CasADi + IPOPT

    ミニカーレースへの応用

    MPCCはミニカーバトルのタイムアタックに特に有効です。

    • コーナー手前で自動的に減速
    • 直線区間では最大加速
    • コース幅を活用した最適ライン選択（アウトインアウト）
    • 障害物回避とタイム最適化の両立
