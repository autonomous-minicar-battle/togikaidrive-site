    判断

    自動運転の2番目のステップ「判断」について学びます。

    判断とは

    判断とは、認知で得た情報をもとに「どう動くか」を決めることです。人間で言えば「考える」に相当します。

    graph LR
    A[認知結果] --> B[判断ロジック]
    B --> C{どう動く?}
    C -->|左に障害物| D[右に曲がる]
    C -->|前に障害物| E[止まる/回避]
    C -->|障害物なし| F[直進]

    ルールベース vs ニューラルネットワーク

    判断方法には大きく2つのアプローチがあります。

    ルールベース

    人間が「こういう時はこうする」というルールを決めておく方法。

    # 例：right_left_3アルゴリズム
if 前方 < 停止距離:
    if 左 > 右:
        左に曲がる
    else:
        右に曲がる
else:
    直進

    メリット: 動作が明確、デバッグしやすい デメリット: 複雑な状況への対応が難しい

    ニューラルネットワーク

    人間の運転データを学習して、AIが判断する方法。

    入力（センサー値） → ニューラルネット → 出力（操作値）
                    ↑
              学習データから自動で獲得

    メリット: 複雑なパターンを学習できる デメリット: 判断理由が分かりにくい

    判断モード一覧

    モード 方式 説明
    right_left_3 ルール 3センサーで障害物回避
    wall_follow ルール 壁沿い走行
    wall_follow_pid ルール PID制御で精密な壁沿い
    nn NN 超音波センサー入力のNN
    donkeycar NN カメラ入力のCNN

    制御方式の比較

    graph TB
    subgraph "ルールベース"
        A1[センサー値] --> A2[if文判定]
        A2 --> A3[操作値]
    end

    subgraph "ニューラルネット"
        B1[センサー値] --> B2[隠れ層1]
        B2 --> B3[隠れ層2]
        B3 --> B4[操作値]
    end

    共通点

    どちらの方式も、最終的な出力は同じ形式：

    • ステアリング: -1.0（左）〜 1.0（右）
    • スロットル: -1.0（後退）〜 1.0（前進）
